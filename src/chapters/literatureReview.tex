\chapter{LITERATURE REVIEW}
Standard sensors struggle to detect people who are not moving. Ahmad et al. \cite{Ahmad2021}  studied hybrid systems that combine PIR and Radar sensors to improve detection in smart buildings. Combining the two reduced false alarms, the system still showed nearly 0\% reliability when occupants were completely still. The study did not explore visual confirmation as a solution. This leaves a gap that computer vision can fill, by detecting the small physical movements that people make even at rest.

Running vision models on small hardware requires significantly reducing input size. Nagel et al. \cite{nagel2021white} examined INT8 Post-Training Quantisation, a technique that converts model weights from 32-bit floating point to 8-bit integers. This reduces memory usage by four times with less than 1\% reduction in accuracy. However, the study does not address the specific constraints of microcontroller based devices like the ESP32-S3, which has only 512 KB of SRAM \cite{esp32s3_datasheet} . Deploying on such hardware requires additional instruction-level optimisations that the paper does not cover.

Wu et al. \cite{wu2016qcnn} explored further compression through the Q-CNN framework. Rather than simply reducing weight size, Q-CNN focuses on matching the output of compressed layers to the original model as closely as possible. This allows models to be compressed by up to 20 times while maintaining strong classification performance. The limitation is that the study only tests single-device deployment and does not address how to process high-resolution images across multiple low-power nodes.

Iandola et al. \cite{iandola2016squeezenet} introduced SqueezeNet, which replaces large convolutional filters with smaller 1×1 layers called Fire Modules. The result is a model that matches AlexNet accuracy using 50 times fewer parametersmaking it well suited for mobile and embedded devices. However, SqueezeNet was not designed for ultra-low-bit operations, which are needed when deploying on the most power-constrained hardware used in this project.

Juefei-Xu et al. \cite{juefei2017lbcnn} took efficiency further with Local Binary Convolutional Neural Networks \gls{lbcnn}. Instead of learned filters, LBCNN uses fixed binary kernels that do n’t change during training process. This reduces the number of parameters by 169 times and speeds up inference in the same way. The drawback is that LBCNN has no builtin way to handle very large resolution images, which are too large to fit in the memory of most microcontrollers.

Li et al. \cite{li2020pytorch}  studied how large images can be broken into grids of smaller sections for processing in deep learning systems. Their findings confirmed that tiling does preserves fine spatial detail that is lost when an image is resized. The study was conducted on GPU clusters, however, and does not provide a communication protocol suited for small, low-power device clusters

Akyon et al. \cite{akyon2022sahi} addressed small object detection directly through the SAHI framework which converts  a high-resolution image into overlapping patches and runs inference on each one separately. This improves detection accuracy for small objects by 20 to 30\% \cite{akyon2022sahi}. The framework is computationally heavy, though, and was designed for powerful workstations rather than real-time edge deployment. In a distributed system, the speed at which devices communicate matters in the same level as accuracy. 

Jeong et al. \cite{jeong2021} studied low-latency communication protocols for clustered IoT devices. Their results showed that peer-to-peer protocols can achieve latency as low as 1.115 ms, which is fast enough for real-time synchronisation. Even though the study did not test image tile transmission specifically, which involves far more data than simple sensor readings.
This project addresses all of these gaps together by deploying a real-time distributed tiling system on ESP32-S3 microcontroller based architecture, combining spatial decomposition, model quantisation, and low-latency communication in a single pipeline designed for resource-constrained hardware.
